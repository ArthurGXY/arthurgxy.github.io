[Hexo] A Brief Introduction 简介

Posted on 2022-01-13 Edited on 2022-04-01 In English , Chinese , Web

What is Hexo?

A fast, simple, powerful blog framework
快速、简洁且高效的博客框架


You write posts in Markdown (or other markup languages) and Hexo generates static files with a beautiful theme in seconds. And you can host it on a cloud server or hosting services like github pages via existing deployer plugins. Great and fast.

可以用 Markdown (或其他标记语言) 编写文章，Hexo 会自动生成静态网页和文件，(还有炫酷的主题)，然后通过部署插件上传到云服务器或者 github pages 之类服务上。多快好省。

Note that hexo accepts shorthand commands, such as hexo v for hexo version and hexo g for hexo generate.
Hexo支持简写的命令，如 hexo v 对应 hexo version，hexo g 对应 hexo generate。

Common hexo arguments
常用 hexo 指令参数

hexo:

Command Short Description
init <folder> i initialize a new Hexo project in <folder>
new [type] n create a new post. Provided other types: title and layout.
generate g generate static files
server s start a local real-time debug server
deploy d deploy to the configured remote server
version v show hexo version
help h show help

Cooperate 合作

If you are going to cooperate with others, you might want to use some sort of version control.
In this case, create a empty reporsitory first, or import the website folders to a existing repository where all of you can share.
如果需要团队合作，可以先创建一个空的仓库，或者将网站文件夹导入到一个已经存在的仓库中来让所有人访问。

Editing 编辑

We have mentioned the command in table 1-1.
新建文章的命令在 表1-1中已经提到了。

The new article created will be in the source\_posts\ folder.
新建的文章会在 source\_posts\ 文件夹中。支持中文标题。

Open that .md file where filename is the same as the title of the article, edit it in Markdown or mixed with HTML, then save it. You can use this during editing:
打开这个 .md 文件，文件名是文章标题，使用 Markdown 或者 HTML 编辑，然后保存。在此期间，你可以用

bash $ hexo server

to check and debug your edit. (Refresh the local website after saving edits)
来实时查看生成效果。（保存编辑后刷新本地网站）

Lastly, use Hexo to generate the static files (generated static files will be under your-project-root\public). Now you have an article to be published.
最后，使用 Hexo 生成静态文件(生成的静态文件在 根目录\public\ 下)。下一步就是发布文章了。
